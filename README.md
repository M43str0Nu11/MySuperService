# MySuperService

## Описание
"MySuperService" — фронтенд-заготовка для будущего набора страниц (десятки и более), с единой цветовой схемой и общей структурой папок. Сейчас реализована шапка с логотипом и раздел "Погода" с примером карточки.

## Структура проекта
```
MySuperService/
├── pages/            # HTML-страницы (index.html и будущие)
├── assets/
│   └── css/
│       └── style.css # Общие стили
└── README.md
```
При необходимости добавьте папки `assets/img/`, `assets/js/`, `assets/fonts/`.

## Цветовая палитра
(Не удаляйте — используется как референс дизайна)
```
#5F94D9  главный акцент (заголовки, контуры, свечение)
#090B0D  фон (основной бэкграунд)
#5A90BF  вторичный цвет / ссылки / вспомогательный текст
#385773  тень акцента / глубина
#F2B366  контрастный акцент (важные значения, подсветка)
```

## Добавление новой страницы
1. Создайте файл в `pages/`, например `about.html`.
2. Базовый шаблон:
```html
<!doctype html>
<html lang="ru">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>About – Pupa & Lupa</title>
  <link rel="stylesheet" href="../assets/css/style.css">
</head>
<body>
  <header class="site-header">
    <span class="logo-title">Pupa &amp; Lupa</span>
  </header>
  <main>
    <!-- Контент страницы -->
  </main>
</body>
</html>
```
3. Используйте существующие классы и цветовую палитру.
4. Новые специфические стили лучше добавлять в отдельный файл `assets/css/<page>.css` и подключать после `style.css` при необходимости.

## Раздел "Погода"
- Заголовок слева, линия тянется вправо и не соприкасается с текстом (`.section-header`, `.animated-line`).
- Карточка погоды оформлена через `.weather-container`, `.weather-current`, `.weather-details`.
- Цвета и свечение выдержаны под общую тему.

## Рекомендации по стилям
- Избегайте инлайновых стилей — вся стилизация через CSS-файлы.
- Для повторяющихся компонентов (виджеты, карточки) используйте общие классы.
- Старайтесь не дублировать цвета — берите их из палитры.

## Очистка и поддержка
- Неиспользуемые временные комментарии удаляйте, но блок палитры оставьте.
- Перед коммитом проверяйте на синтаксические ошибки (VS Code покажет подсветку).

## Дальнейшее развитие
